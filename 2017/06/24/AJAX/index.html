AJAX(Asynchronous Javascript And XML)

异步:

  • 客户端和服务器的数据交互不需要将整个页面刷新,只需要把操作的这一部分刷新,叫做局部刷新
  • AJAX一般都是用来请求页面的部分数据,再将获取出来的数据绑定到页面的指定位置

    AJAX如何使用

    1.创建一个XHR的异步对象(这种方式不兼容IE6及更低版本的浏览器,需要做兼容处理)

    var xhr=new XMLHTTPRequest;

2.打开一个URL地址并请求(发送请求之前的配置)

open([请求方式],[API接口],[设置同步异步],[user name],[User Pass]);

  • 请求方式:get系列:GET/DELETE/HEAD post系列:POST/PUT
  • user name:用户名
  • user Pass:用户密码 有些服务器为了安全考虑只允许部分人访问,就只给部分人分配权限,访问需要安全密钥,一般的服务器不需要这么麻烦,所以只需要传前三个参数即可

    3.监听不同的状态,进行不同业务操作 onreadystatechange

    xhr.readyState:AJAX的状态

    0:unsend 未发送,刚开始创建的一个AJAX的实例 var xhr=new XMLHTTPRequest;
    1:opened 完成了xhr.open();已经打开
    2:headers_received客户端已经收到服务器返回的响应头了
    3:loading 服务器返回的响应主体正在传输
    4:done 服务器返回的响应主体已经被客户端接收了

xhr.status:HTTP状态

通过状态码可以知道这个HTTP事物是否成功,以及失败的原因
[2开头]:代表成功
200:OK
[3开头]:也代表成功了,只不过这个过程经过了特殊处理
301:Moved Permanently 在新版的HTTP协议中代表永久重定向 比如访问http://www.360buy.com 会自动跳转到www.jd.com
302:Moved temporarily 在新版HTTP协议中代表临时重定向 也叫临时转移 ==>服务器的负载均衡
304:Moved Modified获取的是缓存中的数据(网站性能优化的重要手段,我们一般将一些不怎么变的静态资源JS/CSS/IMG等做成304缓存 以后直接在缓存中拿)
[4开头]:代表错误,而且一般都是客户端的错误
400:Bad Request请求参数错了
401:unauthorized 无访问权限
403:Forbidden 请求接收到了,但是拒绝执行此请求,没有原因
404:Not found 请求地址不存在
413:Request Entity Too Large 客户端传给服务器的参数超过了服务器愿意接受的范围 ,服务器拒绝
[5开头] 代表服务器错误
500:Internal Server Error服务器的未知错误
503:Server Unavailable 服务器超负荷

xhr.response:获取响应的主体(一般不用)
xhr.responseText:获取响应主体的内容,是text(字符串)格式
xhr.responseXML:获取响应主体内容是XML格式
xhr.getResponseHeader(“Data”):获取响应头的制定内容
xhr.getAllResponseHeaders;获取所有的的响应头
xhr.timeout:设置请求的超时时间,超过时间就会触发ontimeout事件
xhr.abort():中断当前AJAX请求 一旦终端请求就会触发onabort事件
xhr.setRequestHeader([key],[value]):设置请求头信息

4.send()

AJAX是从send开始的,之前都是在做准备工作,当我们的xhr.readyState===4的时候结束

面试问题

get系列和post有什么区别

1.传参数的方式不同

get通过URL+?+参数
xhr.open(“get”,”/userInfo?name=XXX&age=100”)
xhr.send(null)
post:
xhr.open(“post”,”/userInfo”)
xhr.send(“name-XXX&age=100&sex=1”);

2.get方式容易出现缓存,post不会

原因:GET是通过URL加上?加参数传数据给服务器的,如果参数和上次一样,浏览器就会使用缓存,以为你请求的是同一个url就会把之前的返回给你,这样有些需求是不可以的比如试试获取股票信息就不能走缓存
[清除缓存]原理:保证url每次都不同,这就可以给每个url后面拼接一个变量随机数或者时间戳Math.random();new Date.getTime();

3.参数大小限制 url大小有限,get局限性,传大量参数用post

get是把数据放到url上,但是浏览器的url大小是有限制的,超了自动截断(每个浏览器不一样chrome:8kb firefox:7kb ie:2kb)
post是放在请求主体上面,大小没有限制,但是在真实项目中,为了保证上传速度,一般限制100kb以内

4.安全性,post传参不可见

get不安全,传的参数放在url上面,可见
一般有重要的信息都会采用post请求
