ES6

定义变量

let

  • 没有变量提升,不会预解释,存在死区
  • 不能重复定义
  • 虽然不进行预解释,但是在代码执行之前会把let定义的变量过滤一遍,一旦不合法,直接报错,代码不执行

块级作用域

  • {}包起来的就是一个块级作用域
  • 应用场景,一是原来内部变量可能会覆盖全局变量
  • 二是循环时结束后计数变量i变量泄露变为全局变量
  • {}包起来就是一个自执行函数

const

  • const定义静态变量,不能修改值,必须要赋值;
  • 不是布恩那个修改,基本数据类的话不能修改,如果时引用类,附的是地址,地址内的值可以修改

this问题

箭头函数中没有this,它指向的是定义时所在作用域的this,所以箭头函数的this是固定的,不能通过call,bind,apply等改变

高阶函数

symbol

注意，Symbol函数前不能使用new命令，否则会报错。这是因为生成的Symbol是一个原始类型的值，不是对象
Symbol函数可以接受一个字符串作为参数，表示对Symbol实例的描述，主要是为了在控制台显示，或者转为字符串时，比较容易区分
