JSON

JSON是个对象，是window下的一个属性，也可以理解为是个全局变量，在低版本的IE下没有

JSON与普通对象的区别：

  • JSON对象的属性名用双引号包起来
  • 属性值如果是字符串的话也需要双引号包起来

在IE中不兼容的话使用eval()来实现

兼容：

DOM映射

通过DOM方法获取的一个元素集合（类数组）,这个集合依然和页面保持着联系，并且这个元素集合会随着页面元素的增加而增加，减少而减少

即使把类数组转为一个数组，每个元素仍然和页面有联系，这就叫做DOM映射

回流：

  • 元素的位置发生改变（增加元素，删除元素，移动），会引起回流，让整个页面重新渲染一遍，从而造成性能浪费

重绘：

  • 元素的样式发生改变，就会把当前的这个元素重新渲染一遍

所以以后项目中，能用重回代替的就不用回流，能用一次回流就不要使用多次

向页面中增加元素的方式：

  • 动态创建的方式document。createElement,然后一个个的appendChild，会造成对此回流，这样不好，但是它有个好处是不会改变原来的元素（比如不会影响他之前绑定的事件）
  • 通过拼接字符串，最后再一次性通过innerHTML,这样只会引起一次回流，比较好，但是他会改变原来元素（比如会影响之前绑定的事件）
  • 文档碎片，一个DOM容器，专门来放DOM元素的

文档碎片步骤：

  • 先创建元素
  • 按需求给元素增加一些重要的属性
  • 暂时存放在DOM碎片里面，appendChild进去
  • 循环完之后，把整个文档碎片放到需要添加的元素中，appendChild
